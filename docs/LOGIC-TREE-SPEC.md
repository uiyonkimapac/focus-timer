# 로직트리 기획 캔버스 — 기능 스펙 v1 (Logic Tree Planning Canvas)

*Focus Timer 신규 기능 / 인터뷰 + ADHD-safe 검토 반영 — 2026-06-15*

---

## 한 줄 요약

**이미 존재하는 "막막한 큰 프로젝트(산맥)"를 우클릭해 열고, 손그림 로직트리로 분해해, 그 카테고리에 태스크를 만들어내는 기획 레이어.** List/Map은 *실행*의 진실(source of truth), 트리는 그 위에 얹힌 *기획* 도구. 데스크톱 전용.

> **포지셔닝 (중요):** 트리는 **기본 태스크 생성 경로가 아니다.** Dump Tasks가 정문(빠른 길)이고, 트리는 "쪼개기 막막한 소수의 큰 프로젝트"를 풀 때만 쓰는 특수 도구. 생산적 미루기(예쁜 트리 그리느라 실행 회피)를 막기 위해 의도적으로 좁게 둔다.

---

## 1. 핵심 모델

- **한 트리 = 한 (기존) 카테고리(산맥).** 한 번에 하나의 프로젝트만.
- **루트 = 그 카테고리(고정 라벨, 변환 대상 아님).** 중간 노드 = 구조(그룹·단계). **말단(leaf) = 기본 태스크 후보.**
- 앱의 기존 2단계(카테고리 > 태스크) 평면 구조는 유지. 트리의 다단계는 변환 시 **평면 태스크 + 순서**로 풀린다.

---

## 2. 진입 (Entry) — 데스크톱 전용

- **기존 산맥 우클릭 → 그 산맥의 트리 캔버스 (오버레이 레이어)** 로 열림.
- **신규 카테고리 생성 흐름은 v1 제외.** 기존에 존재하는 카테고리에만 적용.
- **모바일 미적용.** 폰에서 트리 작성은 어렵다 → 데스크톱에서만. (폰은 기존 List/Map 그대로.)

---

## 3. 트리 작성

- **노드 내용 = 이름만.** 시간 입력 없음(시간맹·마찰 방지). 변환된 태스크는 기본 시간으로 생성되고, 필요하면 평소처럼 List에서 조정.
- **조작: 키보드 + 드래그 모두 지원**
  - 마인드맵식 — `Enter` = 형제 노드, `Tab` = 자식 노드, 자동 레이아웃
  - 자유 드래그 배치 + 수동 연결선
- **비주얼: 손그림 지도 톤** — 기존 Mountain Board와 통일된 세계관.

---

## 4. 태스크 변환 (Conversion)

- **선택 변환:** 노드 선택 후 **'태스크로 변환' 버튼**으로 일괄 변환. 기본 추천은 말단(leaf)이지만 어느 노드든 선택 가능.
- 변환된 노드 → 그 산맥(카테고리)의 태스크로. 시간은 기본값.
- **변환은 1회성.** 이후 그 트리는 **읽기전용 보관**(재변환 불가, 추가하려면 새 트리).
- 트리 구조의 순서(부모→자식, 좌→우)는 태스크 **List 정렬 순서로만 암시.** 별도 의존성 UI 없음.

---

## 5. 완료 미러링 (살아있는 진행 지도) ★

- 변환된 태스크를 **완료하면 → 트리의 해당 노드에도 완료 표시(읽기전용 반영).**
- 덕분에 트리는 죽은 그림이 아니라 **프로젝트 진행 현황판**이 된다 — object permanence / working-memory 대신 들어주기라는 앱 정체성과 일치.
- 미러링은 **단방향(태스크 → 트리)** 이고 읽기전용. 트리에서 노드를 고쳐 태스크를 바꾸지는 않음. (구현상 노드가 태스크 id를 가볍게 참조; 태스크가 삭제되면 노드는 그대로 두고 미러만 사라짐.)

---

## 6. 저장 / 라이프사이클

- **산맥당 여러 트리 누적.** 나중에 또 분해하려면 새 트리를 만들어 변환(1회성). 산맥 우클릭 시 그 산맥의 트리 목록을 봄.
- **저장: 기존과 동일** — `localStorage` + 선택적 sync 코드.

---

## 7. MVP (v1) 범위

**포함**
- 기존 산맥 우클릭 → 트리 캔버스 (데스크톱)
- 말단(leaf) → 태스크(이름만, 시간 기본값)
- 선택 변환 버튼 (1회성, 이후 읽기전용)
- 완료 미러링(태스크 → 트리, 읽기전용)
- 키보드 + 드래그 작성, 손그림 톤
- 산맥당 트리 누적, 로컬 + sync

**제외 (의도적으로 v1에서 뺌)**
- 신규 카테고리 생성 흐름 (기존 산맥만)
- 노드 시간 입력
- AI 자동 분해 (인테그레이션 부담)
- 모바일 작성
- 띵킹트리 (자유 발산 모드)
- 의존성의 시각적 표현(잠금·뱃지) — 순서 암시로만

---

## 8. 설계 단계에서 더 정할 것 (비차단)

1. 드래프트·아카이브 트리가 **List 뷰엔 안 보이고 Map/트리 레이어 전용**인지 (가정: 기획은 데스크톱 Map 전용).
2. 아카이브 트리 **노출 방식** — 평소 숨김, 산맥 우클릭 시 목록으로 (권장).
3. 빈 트리 **발견성/온보딩** — 첫 사용자가 "산맥 우클릭"을 알 수 있게. *생산적 미루기 방지를 위해* 변환까지 빠르게 끝나는 동선 우선.
4. v1을 더 단순화: **자유 드래그를 빼고 자동 레이아웃만**으로 시작하는 옵션 (만지작 회피↓) — 추후 결정.

---

## 부록: 합의/검토 반영 로그

- 적용 대상: **기존 산맥만** (신규 카테고리 흐름 제외) · 막막한 큰 프로젝트 전용
- **시간 입력 없음**
- **AI 자동 분해 없음**
- **완료 미러링 채택** (← 기존 "완전 분리"에서 수정)
- **모바일 미적용** (데스크톱 전용)
- 루트 = 기존 카테고리 고정 / leaf → 태스크 / 선택 변환·1회성·읽기전용
- 의존성은 순서로만 암시 · 노드 = 이름만 · 손그림 톤 · 로컬 + sync
